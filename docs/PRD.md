# NewMCP 产品需求文档 (PRD)

> 版本: V1.0 | 状态: 草案 | 更新日期: 2026-05-11

## 1. 产品概述

### 1.1 产品定位
NewMCP 是新一代统一大模型 MCP (Model Context Protocol) 服务管理平台。类似 NewAPI 统一了 LLM API 网关，NewMCP 统一了 MCP 服务的注册、分组、路由和协议桥接。

### 1.2 目标用户
- **主要用户**: 个人开发者 / AI 爱好者（自建 MCP 服务、对接小智设备、控制机器人）
- **次要用户**: 团队/企业（V3 商业化阶段）

### 1.3 核心价值
- **统一管理**: 一个平台管理所有 MCP 服务，不再逐个配置
- **智能分组**: 按场景分组 MCP 服务，LLM 按需发现和调用
- **协议桥接**: 透明转换 Streamable HTTP/WebSocket/被动连接，解决协议碎片化
- **被动接入**: 提供 WSS 接入点，外部 MCP 服务可主动连入注册（类似小智云模式）
- **硬件对接**: 支持主动连接云平台（如小智等），通过 WebSocket 长链接暴露 MCP 工具
- **视觉能力**: 可配置视觉模型 + 摄像头，为 LLM 提供视觉感知

## 2. 功能需求

### 2.1 用户管理
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 注册/登录 | P0 | 用户名 + 密码，JWT 认证 |
| 个人设置 | P1 | 修改密码、头像、基本信息 |
| API Key 管理 | P0 | 创建/删除 API Key，用于 MCP 客户端认证 |
| 角色权限 | P1 | admin/user 角色，admin 可管理所有用户 |

### 2.2 MCP 服务注册中心
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 注册 MCP 服务 | P0 | 支持 5 种传输类型: stdio, SSE, Streamable HTTP, WebSocket, 被动连接 |
| 被动连接 | P0 | NewMCP 生成 WSS 接入点 URL，外部 MCP 服务主动连入注册工具 (passive-ws) |
| 服务配置 | P0 | 配置连接参数（URL/命令/参数/环境变量/认证信息） |
| 自动工具发现 | P0 | 注册后自动调用 `tools/list` 获取工具目录 |
| 手动刷新工具 | P0 | 手动触发工具目录刷新 |
| 健康检查 | P0 | 周期性检查服务可用性，显示状态（健康/不健康/未知） |
| 连接测试 | P0 | 一键测试服务连接是否正常 |
| 服务启用/禁用 | P1 | 不删除，临时禁用服务 |
| 服务标签 | P2 | 给服务打标签，便于分类和搜索 |

**MCP 服务按使用方式分类：**

| 分类 | 说明 | 传输类型 | 使用方式 |
|------|------|----------|----------|
| **即时使用** | 远程托管的 MCP 服务，填写 URL 即可接入 | SSE, Streamable HTTP, WebSocket | 注册时填入远程服务 URL，平台直接连接使用 |
| **自建部署** | 需要本地安装运行的 MCP 服务 | stdio | 注册时配置启动命令和参数，平台在本地启动子进程运行 |
| **被动接入** | 外部服务主动连入平台 | passive-ws | 平台生成 WSS 接入点，外部服务连接注册 |

> 即时使用的 MCP 服务通常由第三方提供商托管（如 Exa Search），用户只需填入 URL 和认证信息即可。
> 自建部署的 MCP 服务需要用户在运行 NewMCP 的机器上安装对应工具（如通过 npm/pip 安装），平台通过 stdio 启动子进程运行。

**MCP 渠道与可见性规则：**

| 角色 | 添加的 MCP 服务 | 可见性 | 市场上架 |
|------|-----------------|--------|----------|
| **普通用户** | 注册个人使用的 MCP 服务 | 仅自己可见（private） | 不可上架 |
| **管理员** | 注册平台级 MCP 服务 | 可设为公开（public），上架至平台市场 | 可上架 |

> 管理员添加的 MCP 服务上架至平台市场后，所有用户可浏览和使用。
> 普通用户添加的 MCP 服务仅限个人使用，其他用户不可见。

**平台市场 MCP 服务分类：**

| 分类 | 说明 | 典型示例 |
|------|------|----------|
| **即用型** | 服务已部署运行，用户可直接接入使用，填入认证信息即可 | Exa Search、Filebase 等远程托管服务 |
| **源码型** | 提供源码和部署文档，用户需下载后在本地自行部署，再注册到平台 | 开源 MCP Server 项目（需 npm/pip 安装后以 stdio 运行） |

> 即用型市场服务：管理员配置好远程 URL，用户从市场一键添加到自己的服务列表，填入 API Key 即可使用。
> 源码型市场服务：管理员上架源码信息（仓库地址、安装说明、配置模板），用户按文档部署后自行注册。

### 2.3 MCP 分组管理
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 创建分组 | P0 | 名称、描述、图标、可见性 |
| 添加/移除服务 | P0 | 将 MCP 服务加入或移出分组 |
| 工具级别控制 | P0 | 在分组内启用/禁用特定工具 |
| 工具选择管理 | P0 | 管理工具面板：按服务分组展示所有工具，支持搜索、批量启用/禁用 |
| 工具重命名 | P1 | 自定义工具在分组内的显示名称 |
| 独立端点 | P0 | 每个分组有独立的 MCP 端点 URL |
| 工具聚合查看 | P0 | 查看分组内所有服务的聚合工具列表 |

### 2.4 MCP 网关
| 功能 | 优先级 | 说明 |
|------|--------|------|
| Streamable HTTP 端点 | P0 | 主传输协议，兼容最新 MCP 规范 |
| WebSocket 端点 | P0 | 支持 WebSocket 传输，用于长链接场景 |
| 双模式暴露 | P0 | Direct 模式（直接暴露所有工具）和 Smart 模式（仅暴露 3 个元工具） |
| Smart 模式搜索引擎 | P0 | BM25 算法搜索 MCP 服务和工具，渐进发现 |
| 工具路由 | P0 | `tools/call` 请求自动路由到正确的上游 MCP 服务 |
| 命名空间隔离 | P0 | 工具名格式 `{ServiceName}__{toolName}` 避免冲突 |
| 协议桥接 | P0 | 客户端和上游可使用不同传输协议 |
| 会话管理 | P1 | 支持有状态/无状态会话模式 |

**Direct 模式 vs Smart 模式:**

| 特性 | Direct 模式 | Smart 模式 |
|------|-------------|------------|
| 暴露工具数 | 全部聚合工具 | 固定 3 个元工具 |
| 工具命名 | `serviceName__toolName` | `serviceName.toolName` (搜索结果) |
| 适合场景 | 工具少（<20） | 工具多（20+），小智等受限设备 |
| Token 消耗 | 高（所有工具 schema） | 极低（仅 3 个元工具 schema） |
| 调用方式 | 一步直接调用 | 搜索→查看→执行 渐进调用 |

Smart 模式暴露的 3 个元工具（参考 mcp-gateway）:
- `mcp.search`: BM25 搜索可用的 MCP 服务和工具
- `mcp.describe`: 查看指定工具的完整 Schema
- `mcp.execute`: 执行指定工具

### 2.5 云端主动连接
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 云端连接管理 | P0 | 统一管理所有主动连接到远端云平台的配置 |
| 多平台支持 | P0 | 支持 xiaozhi、custom(WSS)、ssh 等多种云平台类型 |
| 主动连接远端 | P0 | NewMCP 主动连接远端端点（WSS/SSH），作为 MCP Server 向远端注册工具 |
| 绑定 API Key | P0 | 每个连接绑定一个 API Key，通过 API Key 的权限决定向远端暴露哪些 MCP 服务和分组 |
| 断线重连 | P0 | 连接断开自动重连（指数退避），重连后自动重新注册工具 |
| 连接状态监控 | P1 | 前端实时显示连接状态（已连接/断开/错误） |
| Token 过期提醒 | P1 | 显示 Token 过期时间，提前提醒用户更新 |
| 多连接支持 | P1 | 支持配置多个云端连接（对应不同平台/Agent），绑定不同 API Key |

> **API Key 绑定说明**: 云端连接绑定 API Key 后，远端设备可使用的 MCP 工具范围由该 API Key 的权限配置决定（复用现有 API Key 的 `permissions.groups` 逻辑）。例如 API Key 配置了 `groups: ["robot-control"]`，则该云端连接只能暴露 robot-control 分组内的工具。

### 2.6 视觉 MCP 服务
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 视觉模型配置 | P0 | 配置模型提供商（OpenAI/Ollama/自定义）、API Key、端点 URL |
| 模型测试 | P0 | 一键测试视觉模型是否可用 |
| 自动注册 MCP | P0 | 配置完成后自动注册为虚拟 MCP 服务 |
| 视觉工具 | P0 | `vision.analyze_image`, `vision.describe_scene` |

### 2.7 外挂视觉系统（摄像头）
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 摄像头配置 | P0 | 配置摄像头源（HTTP MJPEG/RTSP/本地设备） |
| 帧率设置 | P0 | 可配置截图帧率（FPS） |
| 分辨率设置 | P1 | 可配置捕获分辨率 |
| 绑定视觉模型 | P0 | 将摄像头绑定到已配置的视觉模型 |
| 手动捕获 | P0 | 手动触发截图 |
| 摄像头 MCP 工具 | P0 | `camera.capture`, `camera.analyze`, `camera.get_latest` |

### 2.8 管理与监控
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 调用日志 | P1 | 记录所有 MCP 工具调用（请求/响应/耗时） |
| 统计仪表盘 | P1 | 调用量、成功率、平均延迟等 |
| 限流 | P1 | 按 API Key / 服务级别限流 |
| 错误告警 | P2 | 服务异常时通知 |

### 2.9 社区 MCP 市场（后期规划）
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 市场浏览 | P1 | 所有用户可浏览、搜索平台市场的 MCP 服务（即用型 + 源码型） |
| 一键添加（即用型） | P1 | 从市场一键将即用型服务添加到个人服务列表，填入 API Key 即可 |
| 源码部署引导（源码型） | P1 | 展示仓库地址、安装说明、配置模板，引导用户自行部署 |
| 提交上架申请 | P2 | 用户可提交自制的 MCP 服务包（代码+配置+文档），申请上架至市场 |
| 审核流程 | P2 | 管理员审核 + 自动安全扫描，审核通过后上架 |
| 评分评论 | P3 | 用户对市场服务评分和评论 |
| 收藏功能 | P3 | 收藏感兴趣的服务 |
| 版本管理 | P3 | 服务版本管理和更新推送 |

## 3. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 单实例支持 100+ 并发 MCP 连接 |
| 可靠性 | 上游 MCP 服务故障时自动标记，不影响其他服务 |
| 安全 | API Key + JWT 双认证，敏感信息加密存储 |
| 可部署 | 单二进制 + SQLite 可运行，也支持 MySQL/Redis |
| 兼容性 | 完全兼容 MCP 协议规范 2025-03-26 及以上版本 |
| 多语言 | 前端支持中文/英文 |

## 4. 版本规划

### V1（核心功能，约 12 周）
- MCP 服务注册中心 + 分组管理 + 网关
- 协议桥接（4 种传输协议）
- 小智设备集成
- 视觉 MCP + 摄像头
- 基础管理和监控

### V2（物联网扩展）
- 本地 MCP 直连转发
- IoT 参数配置（自动发现和启动 MCP 服务）
- 智能家居协议支持（MQTT/Home Assistant/Zigbee2MQTT）
- 机器人控制协议扩展

### V3（商业化）
- 多租户支持
- 付费体系（公共 MCP 市场 + 自建 MCP）
- 支付集成（微信/支付宝/Stripe）
- 用量计量和配额管理
- 企业级功能（SSO/审计日志）

### V4（社区生态，后期规划）
- **社区 MCP 市场**
  - 用户可提交自制的 MCP 服务包（代码+配置+文档）申请上架
  - 管理员审核 + 自动安全检测，通过后上架至市场
  - 市场服务分两类：即用型（直接接入）和源码型（下载自建）
  - 其他用户可浏览、搜索、一键添加（即用型）或按文档部署（源码型）
  - 服务评分、评论、收藏功能
  - 服务版本管理和更新推送
- **开发者激励**
  - 服务使用量排行
  - 优质服务推荐/精选

## 5. 用户故事

### US-01: 注册和使用 MCP 服务
> 作为开发者，我想注册一个 exa 搜索 MCP 服务到平台，这样我的 Claude Code 就可以通过平台统一端点使用搜索功能。

### US-02: 创建分组管理机器人
> 作为机器人爱好者，我想创建"机器人控制"分组，把海陆空三种机器人的 MCP 服务加入其中，这样小智设备连接后就能统一控制所有机器人。

### US-03: 配置视觉能力
> 作为开发者，我想配置 OpenAI Vision API 作为视觉 MCP，并绑定一个 RTSP 摄像头，这样 LLM 就能实时看到摄像头画面并分析。

### US-04: 云端主动连接
> 作为用户，我想在 NewMCP 配置一个云端连接（如小智云平台），粘贴平台提供的 WSS 链接。NewMCP 主动连接远端平台，将我注册的 MCP 工具（如 calculator、搜索）暴露给远端设备。

### US-05: 任务自主执行
> 作为开发者，我想让 LLM 同时使用实时数据 MCP 和机器人控制 MCP，这样 LLM 能根据实时数据分析自动调整机器人指令，直到任务完成。
